TravelMate ID

AI Development • Semantic Search Engine

Role

AI Engineer & Fullstack Dev

Timeline

Dec 2025

Tech Stack

Python, Flask, BERT, FAISS

The Challenge

Traditional travel search engines rely heavily on keywords, often failing to understand context like "healing spots" or "adrenalin activities".

Furthermore, tourism data in Indonesia is often fragmented. Location data exists in one dataset, while pricing and ratings are in another. A major technical hurdle was AI Hallucination—where searching for "Malang" could incorrectly return results for "Pemalang" due to string similarity.

The Solution

TravelMate ID is a smart tourism recommendation system powered by Hybrid Semantic Search. I combined Strict Regex Filtering to lock specific locations (preventing hallucinations) with Sentence-Transformer (BERT) to understand the semantic meaning of user queries.

To solve the data fragmentation, I implemented a Smart Data Enrichment pipeline using Python Pandas to merge datasets using fuzzy matching logic. The system also intelligently imputes missing price/rating data based on the tourism category to ensure a seamless user experience.

Key Features

Hybrid AI Search

Combines Regex for accurate location filtering with Vector Search (FAISS) for context.

Smart Data Merge

Unified multiple datasets to provide comprehensive data (Price + Rating + Location).

Contextual Tips

Auto-generates outfit & weather advice based on the destination type (e.g., Hiking vs Beach).
